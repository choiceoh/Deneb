# 모닝레터

매일 아침 발송하는 일일 브리핑 레터.

## 실행 조건

- 크론잡 `morning-letter`가 매일 08:00 KST에 자동 트리거
- 사용자가 "모닝레터", "아침 브리핑" 등을 요청할 때 수동 실행
- 기본 모델: 게이트웨이의 메인 모델을 자동 사용 (크론잡 payload에 model 미지정 시 defaultModel 폴백)

## 필수 환경변수

- `METALPRICEAPI_KEY`: MetalpriceAPI 무료 키 (구리시세 조회용)

## 실행 절차

### 1단계: 데이터 수집

`morning_letter` 도구를 호출한다. 파라미터 없이 호출하면 5개 섹션을 병렬 수집하여 JSON 데이터를 반환한다.

```json
{"tool": "morning_letter"}
```

반환 데이터 구조:
- `date`: 오늘 날짜 (한국어, 요일 포함)
- `sections.weather`: 기온, 체감온도, 날씨상태, 습도, 최저/최고, 강수확률
- `sections.exchange`: USD/KRW, EUR/KRW 환율 (숫자)
- `sections.copper`: LME 구리시세 USD/ton (MetalpriceAPI)
- `sections.calendar`: 오늘 일정 목록
- `sections.email`: 전일 수신 메일 목록 (발신자, 제목, 스니펫)

각 섹션에 `ok: false`이면 해당 섹션 조회에 실패한 것이다.

### 2단계: 레터 조율 및 작성

도구가 반환한 데이터를 아래 템플릿과 규칙에 따라 **한국어로** 정리하여 하나의 완성된 메시지를 작성한다.

#### 템플릿

```
🌅 모닝레터 — {date}

☁️ 날씨 (광주광역시)
{현재 기온, 체감, 날씨상태 요약}
{최저/최고, 습도}
{강수확률 30% 이상이면 우산 권장 한마디}

💱 환율
USD/KRW: {매매기준율, 천단위 콤마}
EUR/KRW: {매매기준율, 천단위 콤마}

🔶 구리시세
LME: ${가격}/ton
{date 필드로 데이터 날짜 표시}

📅 오늘 일정
{일정 목록을 시간 순으로 정리, 최대 8건}
{일정 없으면 "일정 없음"}

📧 전일 메일 주요사항
{중요도/발신자 기준 상위 5건 선별}
{각 메일: 발신자명 — 제목 1줄 요약}
{메일 없으면 "수신 메일 없음"}

좋은 하루 되세요!
```

#### 조율 규칙

- **날씨 해석**: 단순 숫자 나열이 아니라 맥락적 한마디 추가 (예: 강수확률 높으면 "우산 챙기세요", 한파면 "따뜻하게 입으세요")
- **구리시세**: `price_per_ton_usd` 값을 `$X,XXX.XX/ton` 형식으로 포맷. `date` 필드가 오늘이 아니면 "(X월 X일 기준)" 명시
- **메일 큐레이션**: 전체 목록에서 중요도 순으로 5건 선별. 발신자명은 "이름 <이메일>" → "이름"만 표시. 제목이 길면 핵심만 요약
- **일정 정리**: gcalcli 원문을 깔끔한 `시간 — 제목` 형식으로 정리
- **실패 섹션**: `ok: false`인 섹션은 "조회 실패" 또는 "조회 불가"로 표시
- **전체 실패**: 모든 섹션이 실패해도 날짜와 인사말이 포함된 기본 레터 작성

#### 포맷 규칙

- 한국어로 작성
- 숫자에 천단위 콤마 (예: 1,350.50)
- 환율은 소수점 2자리
- 구리시세는 소수점 2자리 (USD/ton)
- 각 섹션 3줄 이내로 간결하게
- 총 4000자 이내 (텔레그램 제한). 초과 시 메일 요약을 3건으로 축소
- 중간 상태 메시지 없이 최종 레터만 출력
