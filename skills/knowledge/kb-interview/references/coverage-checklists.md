# 커버리지 체크리스트 (질문 은행)

doc 타입별 **반드시 채워야 할 항목 + 그걸 끌어낼 질문**. 그릴 루프는 이 빈 칸을
메우는 게 목표다. 항목이 unknown이면 명시적으로 그렇게 둔다. 모호한 답은 옆의
"파고들기"로 구체화한 뒤 기록한다.

> 이 지식은 모델·웹에 없는 신선한 사내 정보다 → **작성일·출처·확신도 필수**,
> 추측 금지(unknown은 unknown으로). 모두 wiki 페이지로 기록하고 갱신은 supersedes.

## 경쟁사 (competitor)

| 항목 | 끌어낼 질문 | 파고들기 |
|---|---|---|
| 정체/소속 | 회사명·법인·계열? 주력 사업? | "규모?"→매출/인원 숫자로 |
| 제품·서비스 | 우리와 겹치는 라인? 핵심 SKU/서비스? | "비슷"→정확히 어떤 제품이 경합 |
| 가격·거래조건 | 가격대가 우리 대비? 결제·납기 조건? | "싸다/비싸다"→몇 %·얼마 |
| 타깃·포지셔닝 | 누구에게 파나? 어떤 이미지로? | "일반"→어떤 고객군 구체적으로 |
| 강점 | 그들이 우리보다 잘하는 것? | "좋다"→무엇이 어떻게 우위인지 |
| 약점 | 그들의 빈틈·고객 불만·한계? | "별로"→구체적 사례·근거 |
| 우리 차별점 | 우리가 이기는 지점? 왜 고객이 우리를? | 막연하면 실제 수주 사례로 |
| 채널·핵심관계 | 유통망? 핵심 고객·파트너? | 이름·관계 성격 |
| 출처/확신도 | 이 정보 어디서? 얼마나 확실? | 직접경험/전언/공개자료 구분 |

★ 자주 변하는 항목(수주·분위기·인사)도 wiki에 적되 **작성일·확신도를 꼭 달고**, 바뀌면 supersedes로 새 버전을 잇는다.

## 시장 세분 (market-segment)

| 항목 | 끌어낼 질문 | 파고들기 |
|---|---|---|
| 세분 기준(축) | 무엇으로 나누나? (지역/규모/용도/고객유형…) | 축이 여러 개면 1차/2차 구분 |
| 세그먼트 목록 | 그 축으로 나오는 칸들은? | 빠진 칸 없나 되짚기 |
| 각 규모/성장 | 세그별 크기·성장세? (대략이라도) | "크다"→상대비중·추세 |
| 각 니즈/특성 | 세그별 핵심 니즈·구매 기준? | "품질 중시"→어떤 품질 지표 |
| 우리 적합도 | 우리가 잘 맞는 세그? 약한 세그? | 왜 맞나/안 맞나 근거 |
| 진입 우선순위 | 어디부터? 그 근거는? | 직감인지 데이터인지 |
| 출처/확신도 | 데이터 기반? 감? | 확신 낮은 칸 표시 |

## 거래처 (partner) — work-kb partners 챕터용

| 항목 | 끌어낼 질문 |
|---|---|
| 분류 | 공급사/시공사/유통/관청 중 무엇? |
| 거래조건 | 표준 결제·납기·마진 규칙? |
| 계약형태 | 단가계약/건별/MOU? |
| 담당팀 | 사내 어느 팀이 맡나? |
| 이력 배경 | 거래 시작 경위·관계 성격(안정분만)? |

## 프로세스 (process)

| 항목 | 끌어낼 질문 |
|---|---|
| 단계 | 시작~끝 단계를 순서대로? |
| 각 산출물 | 단계별 결과물·통과 게이트? |
| 결재/승인 | 누가 언제 승인하나? |
| 소요/SLA | 단계별 기간? |
| 필수서류 | 단계별 필요한 문서? |
| 예외/리스크 | 자주 막히거나 틀어지는 곳? |

## generic (그 외 모든 토픽)

정의/범위 · 핵심 사실 · 분류/구조 · 기준/규칙 · 예외 · 관계(다른 것과의 연결) · 출처/확신도.
각 항목에서 모호하면 숫자·이름·예시·날짜로 구체화한다.
