# 주간업무보고

기획조정실 주간업무보고를 위키 프로젝트 데이터로 자동 생성한다. `/weekly`(또는 크론)가
트리거하면 게이트웨이가 **직접** 위키를 스캔해 소관별 실시/예정을 모으고 격자 양식 PDF까지
렌더한 뒤, 에이전트에게는 전달만 맡긴다. 이건 **작성 절차서가 아니라 전달 자세**다.

## 실행 조건

- `/weekly` 수동 실행, 또는 크론잡(**매주 토요일 08:00 KST** — `/weekly` 메시지로 트리거).
- 윈도우: 오늘 기준 **실시=지난주(전주, 월~일)**, **예정=이번주(월~일)**. 자동 계산된다(토요일 실행 시 실시=전주).
- 소관(팀) 분류는 프로젝트 위키의 `소관:<팀>` 태그(`1팀`/`2팀`/`3팀`/`남도에코`/`개인`)에 따른다.

## 전달 자세

게이트웨이가 메시지에 주입한 안내(`[주간업무보고 ...]`)를 그대로 따른다:

- **PDF가 생성된 경우** — `send_file` 도구(`type: document`)로 그 PDF를 사용자에게 보내고,
  소관별 건수와 임박 마감 등 핵심만 **2~3줄로 요약**해 덧붙인다. 보고서 전체를 텍스트로
  다시 풀어쓰지 마라(PDF가 본문이다).
- **PDF 렌더 불가(메모리 여유 부족)인 경우** — 주입된 텍스트 보고서를 그대로 깔끔히 정리해
  최종 텍스트로 전달한다. 이때도 도구를 새로 호출하지 마라.

## 주의

- **위키에 있는 사실만.** 숫자·날짜·상태를 지어내지 마라 — 게이트웨이가 이미 위키에서 뽑아
  실시/예정 줄을 합성해 두었다.
- 소관 분류는 프로젝트 페이지의 `소관:` 태그 기반이며, `email-analysis`·`meeting-minutes`가
  페이지를 갱신할 때 자동으로 붙인다. 태그가 빠진 페이지는 보고서에서 누락되니, 보이지 않는
  프로젝트가 있으면 해당 위키 페이지에 `소관:` 태그가 있는지 확인하라.
- 채널 상태 추측("텔레그램 연결 안 됨" 등)·내부 토큰(`<thinking>`, `NO_REPLY`) 노출 금지.
  빈 서두 없이 바로 본론.
