📖 Git 용어 사전

Git에서 자주 쓰이는 용어들을 쉽게 설명합니다. 대시보드에서 점선 밑줄이 있는 단어에 마우스를 올려도 설명을 볼 수 있어요.

📦 기본 개념

저장소 (Repository) repository
프로젝트의 모든 파일과 변경 기록이 담긴 폴더예요. 이 대시보드가 보여주는 것이 바로 하나의 저장소입니다.
저장 (커밋) commit
변경 사항을 기록하는 행위예요. 게임의 "세이브 포인트"와 비슷합니다. 커밋할 때마다 "무엇을 왜 바꿨는지" 메시지를 남깁니다.
준비 (스테이징) staging
저장하기 전에 "이 파일을 포함시키겠다"고 고르는 단계예요. 장바구니에 물건을 담는 것처럼, 저장하고 싶은 변경만 골라 담을 수 있어요.

🌿 갈래 & 합치기

작업 갈래 (브랜치) branch
독립적인 작업 흐름이에요. 나뭇가지처럼 본 줄기에서 갈라져 나와 독립적으로 작업한 뒤, 다시 합칠 수 있어요. "main"이 본 줄기이고, 새 기능은 새 갈래에서 만듭니다.
합치기 (머지) merge
두 갈래의 작업을 하나로 합치는 것이에요. 새 기능 갈래에서 완성된 작업을 main 갈래에 합치면, 그 기능이 본 프로젝트에 반영됩니다.
풀 리퀘스트 (PR) pull request
"내 작업을 합쳐주세요"라고 요청하는 것이에요. GitHub에서 코드 변경 내용을 검토하고 토론한 뒤 합칩니다.

🔄 동기화

밀어올리기 (푸시) push
내 컴퓨터의 저장 기록을 GitHub(인터넷)에 올리는 것이에요.
내려받기 (풀) pull
GitHub의 최신 기록을 내 컴퓨터로 가져오는 것이에요. 다른 사람이 올린 변경 사항을 받아올 때 사용합니다.

📊 이 대시보드에서 보이는 것들

해시 (Hash) 예: 81aa1a0c
각 저장 기록의 고유 ID예요. 사람의 주민등록번호처럼, 모든 커밋에는 절대 겹치지 않는 고유 번호가 있습니다. 대시보드에서 파란 코드로 표시됩니다.
변경 내용 (Diff) diff
두 버전 사이에 무엇이 달라졌는지 보여주는 화면이에요. 초록색(+)은 추가된 줄, 빨간색(-)은 삭제된 줄입니다.
+45 -12 같은 숫자
45줄이 추가되고 12줄이 삭제되었다는 뜻이에요. 꼭 45줄을 "새로 쓴" 것은 아니고, 기존 줄을 고치면 "삭제 후 추가"로 세어집니다.
