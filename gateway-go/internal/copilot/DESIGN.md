# Copilot: 게이트웨이 자율 조정 AI

## 개념

Copilot은 단순 모니터가 아니라 **게이트웨이 런타임에 직접 개입하여 동작을 조정하는 AI**다.

게이트웨이 내부 이벤트(프로바이더 장애, 컴팩션 실패, context overflow)를 실시간으로 관측하고,
문제 발생 시 사람의 개입 없이 스스로 판단하여 프로바이더 전환, 컴팩션 전략 변경 등을 수행한다.

## 아키텍처

```
게이트웨이 런타임 (chat/run.go, compaction.go)
    │ 이벤트 발생
    ▼
Tracker ─── 수집 ───→ Advisor ─── 판단 ───→ 게이트웨이에 개입
                         │
                         ▼
                   Telegram 알림
```

- **Tracker**: 프로바이더 호출 성공/실패, 컴팩션 결과, 폴백 이벤트를 in-memory 수집
- **Advisor**: Tracker 데이터를 보고 개입 여부를 판단. 프로바이더 강제 전환, 컴팩션 파라미터 조정
- **Service**: 주기적 시스템 체크 + Advisor 판단 결과를 텔레그램으로 알림

## 자동 개입 시나리오

**프로바이더 장애 → sglang 자동 전환**:
프로바이더 연속 실패 감지 → sglang으로 강제 라우팅 → 쿨다운 후 원래 프로바이더 복원 시도

**컴팩션 반복 실패 → 전략 자동 조정**:
컴팩션 실패율 상승 감지 → threshold/freshTailCount 파라미터를 공격적으로 조정

## 현재 상태

현재는 외부 시스템 체크(디스크, GPU, 프로세스)만 수행하는 수동 모니터.
Tracker/Advisor는 미구현. 게이트웨이 런타임과 연결되어 있지 않음.
